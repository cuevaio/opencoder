# Better Auth Tables With Drizzle Push

## Goal
Make `drizzle-kit push` keep/create Better Auth core tables by moving table ownership into Drizzle schema.

## Why this happened
- `drizzle.config.ts` points to `src/db/schema.ts` as the single schema source.
- Better Auth tables (`user`, `account`, `session`, `verification`) are not defined in `src/db/schema.ts`.
- `drizzle-kit push` treats missing-from-schema tables as drift and can remove them.

## Recommended implementation
1. Add Better Auth core tables to Drizzle schema.
   - File: `src/db/schema.ts`
   - Add exports using non-colliding symbol names:
     - `authUsers` -> table `"user"`
     - `authAccounts` -> table `"account"`
     - `authSessions` -> table `"session"`
     - `authVerifications` -> table `"verification"`
   - Use Better Auth default column names/types (camelCase DB columns), including `accessToken`, `providerId`, `userId`, `emailVerified`, `expiresAt`, etc.
   - Add required unique/index constraints used by Better Auth lookups:
     - `user.email` unique
     - `session.token` unique
     - `account(providerId, accountId)` unique
     - indexes on `account.userId`, `session.userId`, `verification.identifier`
   - Keep existing app table `agent_sessions` unchanged; avoid naming collisions by only using `authSessions` for the TS export.

2. Keep Better Auth runtime config unchanged.
   - File: `src/lib/auth.ts`
   - No table remapping should be needed if table and column names match Better Auth defaults.

3. Update comments/docs to reflect new ownership model.
   - File: `src/db/schema.ts` (note that Drizzle now owns Better Auth core tables)
   - File: `src/lib/auth-helpers.ts` (replace “auto-created” wording)
   - File: `README.md` (replace `@better-auth/cli migrate` as primary path in this repo with Drizzle migration flow)

4. Create and apply migration.
   - Run `npm run db:generate` to create a new migration for auth tables.
   - Review migration SQL before applying; ensure it creates auth tables and does not include unintended drops.
   - Apply with `npm run db:migrate` (preferred for shared/prod-like envs). Use `npm run db:push` for local/dev reconciliation.

## Environment handling
- If auth tables were dropped already, recreated tables will be empty.
- Existing users may need to sign in again unless table data is restored from backup.
- If any env still has old Better Auth-managed tables, migration should be reviewed for compatibility before rollout.

## Critical files to modify
- `src/db/schema.ts`
- `src/lib/auth-helpers.ts`
- `README.md`

## Verification
1. `npm run db:generate`
2. Inspect generated migration in `drizzle/*.sql` for expected auth table DDL and no unintended destructive statements.
3. Apply schema:
   - `npm run db:migrate` (preferred), or `npm run db:push` in dev.
4. Confirm DB objects exist (`user`, `account`, `session`, `verification`) with expected columns.
5. Run app checks:
   - `npm run check`
   - `npm run test`
6. Run an end-to-end auth sanity check:
   - Sign in with GitHub.
   - Hit the repos endpoint flow that reads from `account.accessToken` (`src/routes/api/github/repos.ts`).

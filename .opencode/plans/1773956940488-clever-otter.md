# Stream Completion Reload Regression Plan

## Goal
Fix the regression where chat appears to reload/remount when a message finishes streaming.

## Root Cause Summary
1. `src/routes/_authed/chat.tsx` currently renders **two `<Outlet />` trees** (desktop + mobile), with one hidden by CSS. Hidden trees still mount, causing duplicate subscriptions/effects and remount-like behavior.
2. `src/components/chat/ChatView.tsx` returns a full loading skeleton whenever `sessionRow` is temporarily missing. During terminal sync updates, this can flash and look like a page reload.
3. `src/routes/_authed/chat.$sessionId.tsx` forces remounts with `key={sessionId}` on `<ChatView />`.

## Implementation Plan

### 1) Render a single route subtree in chat layout
- **File:** `src/routes/_authed/chat.tsx`
- Refactor layout so only **one `<Outlet />`** is mounted.
- Keep responsive UI behavior (desktop inline sidebar, mobile Sheet sidebar), but do not duplicate the child route tree.
- Keep existing `sessionsCollection` sharing and sidebar context behavior.

### 2) Remove forced remount key on chat session page
- **File:** `src/routes/_authed/chat.$sessionId.tsx`
- Remove `key={sessionId}` from `<ChatView />`.
- Preserve current props and navigation behavior.

### 3) Make ChatView resilient to transient session row gaps
- **File:** `src/components/chat/ChatView.tsx`
- Introduce a stable `effectiveSessionRow` derived from:
  - latest non-null session row from live query, or
  - last known non-null row cached in `useRef`.
- Show loading skeleton **only on initial load** (no current row and no cached row).
- Use `effectiveSessionRow` for `initialPrompt`, `sessionStatus`, repo metadata, totals, and cancel run id.

## Files To Modify
- `src/routes/_authed/chat.tsx`
- `src/routes/_authed/chat.$sessionId.tsx`
- `src/components/chat/ChatView.tsx`

## Verification
1. Run `npm run dev`.
2. Reproduce flow in `/chat/$sessionId`:
   - start prompt,
   - wait until stream completes,
   - confirm no full-view flash/reload and no skeleton flash.
3. Confirm no extra document navigation request at completion in browser network panel.
4. Repeat on desktop and mobile viewport.
5. Switch between sessions and verify chat state updates correctly without unintended remount behavior.
6. Run `npm run build` and `npm run check`.

## Success Criteria
- End-of-stream transition is visually stable.
- Chat subtree is mounted once per active route.
- No apparent reload/remount when status changes to idle/completed.

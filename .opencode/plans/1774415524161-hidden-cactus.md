# Mobile Theme Crash Hotfix Plan

## Goal
Fix the mobile Safari crash that happens when switching theme (reported on `/chat/:sessionId`) with a stability-first patch.

## Working Diagnosis
- Theme switching toggles `.dark` on `<html>` and triggers a full-app repaint.
- Mobile chat pages currently combine that repaint with blur-heavy overlays/bars (`backdrop-blur`) and animated sheets.
- This is a high-risk compositing path on iOS Safari and is the most likely crash trigger.

## Implementation Plan (Recommended)
1. Remove blur-heavy compositing on mobile/chat surfaces.
   - Update `src/components/ui/sheet.tsx` to remove overlay blur (`backdrop-blur-[1px]`) and keep a plain dim overlay.
   - Update `src/components/chat/ChatView.tsx` to remove `backdrop-blur-sm` from chat header/footer bars.
   - Update `src/routes/_authed/chat.index.tsx` to remove `backdrop-blur-sm` from the mobile header.
   - Update `src/components/app-header.tsx` to remove header `backdrop-blur-sm` (shared mobile entry point).

2. Reduce motion/compositor pressure during sheet interactions.
   - In `src/components/ui/sheet.tsx`, reduce sheet open/close animation durations on mobile to make theme changes less likely to overlap expensive transitions.

3. Harden theme application path against unnecessary DOM work.
   - In `src/lib/theme.ts`, make `applyThemeToDocument(theme)` idempotent (no class mutation when already in target state).
   - In `src/components/theme-toggle.tsx` + mobile menu call sites (`src/components/chat/ChatMobileMenu.tsx`, `src/components/app-header.tsx`), close the mobile sheet first, then apply theme on the next frame so theme repaint does not occur while overlay animation is active.

4. Add Safari compatibility fallback in media-query hook.
   - In `src/hooks/use-media-query.ts`, keep `addEventListener/removeEventListener` for modern browsers but fall back to `addListener/removeListener` for older Safari behavior.

5. Add/adjust tests for changed logic.
   - Extend `src/lib/theme.test.ts` to cover idempotent application behavior.
   - Add a targeted test for `use-media-query` fallback behavior if practical; otherwise validate manually in Safari.

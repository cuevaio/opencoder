# Plan: Delete Session Feature

## Summary

Add the ability to delete a session from the chat view header. Deleting a session removes all related data from the database (session events, git state are cascade-deleted). If the session is still running, it is cancelled first via Trigger.dev before deletion. A confirmation dialog prevents accidental deletion.

## Changes

### 1. Backend: Add DELETE handler to `src/routes/api/agent/sessions.$id.ts`

Add a `DELETE` handler alongside the existing `GET` handler:
- Call `requireAuth(request)` to get the authenticated user
- Extract session ID from URL path (reuse existing pattern)
- Query the session row to verify ownership (`userId` match) and check status
- If the session is running (`status === "running"`), cancel it first by setting `cancelRequested` metadata on the Trigger.dev run (same logic as `cancel.ts`)
- Delete the session: `db.delete(sessions).where(and(eq(sessions.id, sessionId), eq(sessions.userId, userId)))`
- Cascade constraints on `sessionEvents` and `agentSessionGitState` handle child row cleanup automatically
- Return `{ ok: true }` on success

### 2. Frontend: Add delete button to ChatView header — `src/components/chat/ChatView.tsx`

In the header bar's right-side `<div>` (line 473), add a trash icon button next to `ChatMobileMenu`:
- Import `Trash2` from `lucide-react`
- Add a new `onDeleteSession` callback prop to `ChatViewProps`
- Render a small icon button with `Trash2` icon in the header's right actions area
- On click, open a confirmation dialog

### 3. Confirmation dialog component — `src/components/chat/DeleteSessionDialog.tsx`

Create a small dialog component using the existing `Dialog` UI primitives from `src/components/ui/dialog.tsx`:
- Props: `open`, `onOpenChange`, `onConfirm`, `isDeleting`
- Content: warning text explaining the action is permanent, a Cancel button, and a destructive Confirm button
- The destructive button uses `variant="destructive"` from the existing Button component

### 4. Wire up delete in route page — `src/routes/_authed/chat.$sessionId.tsx`

Add a `handleDeleteSession` callback in `ChatSessionPage`:
- Set loading/error state
- `fetch` DELETE to `/api/agent/sessions/${sessionId}`
- On success: invalidate `["sessions"]` query via `useQueryClient()`, then `navigate({ to: "/chat" })`
- On error: show error in the existing error display
- Pass `onDeleteSession` and `isDeleting` to `ChatView`

### 5. Update ChatView props

- Add `onDeleteSession: () => void` and `isDeleting?: boolean` to `ChatViewProps`
- Manage dialog open state locally in ChatView
- Wire trash button -> dialog -> `onDeleteSession`

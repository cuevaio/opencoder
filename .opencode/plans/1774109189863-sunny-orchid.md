# Fix chat mobile controls and hamburger parity

## Goal
- Stop the mobile `X` and `+` control collision in the chat sessions drawer.
- Make the mobile sidebar trigger on both `/chat` and `/chat/:sessionId` match the same hamburger pattern used on other pages (`AppHeader`).

## Recommended implementation
1. **Remove the conflicting drawer close icon in chat mobile sheet**
   - **File:** `src/routes/_authed/chat.tsx`
   - Update the mobile `SheetContent` to set `showCloseButton={false}`.
   - Keep the existing `SessionSidebar` header `+` button unchanged.
   - Result: no overlap between sheet-level absolute `X` and sidebar header `+`.

2. **Unify `/chat` mobile trigger with app hamburger style**
   - **File:** `src/routes/_authed/chat.index.tsx`
   - Replace `PanelLeftOpen` import/usage with `Menu` from `lucide-react`.
   - Update the mobile toggle button classes to match `src/components/app-header.tsx` hamburger styling (size, icon size, colors, hover).
   - Keep behavior unchanged: click opens the same left sessions drawer via `setSidebarOpen(true)`.

3. **Unify `/chat/:sessionId` mobile trigger with app hamburger style**
   - **File:** `src/components/chat/ChatView.tsx`
   - Replace `PanelLeftOpen` import/usage with `Menu`.
   - Apply the same button classes used in step 2 so both chat routes are visually and behaviorally consistent.
   - Keep the existing header structure and all chat/session logic unchanged.

## Critical files to modify
- `src/routes/_authed/chat.tsx`
- `src/routes/_authed/chat.index.tsx`
- `src/components/chat/ChatView.tsx`

## Verification
1. **Manual UI checks (mobile widths)**
   - Run app and test `/chat` and `/chat/:sessionId` at ~320px, 375px, and 390px widths.
   - Confirm hamburger appears on both routes and opens the left sessions drawer.
   - Confirm no overlap in drawer header (`Sessions` + `+` only; no colliding `X`).
   - Confirm drawer still closes via overlay click/gesture and route transitions.

2. **Desktop regression check**
   - Confirm desktop sidebar and headers are unchanged on both routes.

3. **Project checks**
   - Run `npm run check`.
   - Run `npm run build` for a quick integration sanity check.

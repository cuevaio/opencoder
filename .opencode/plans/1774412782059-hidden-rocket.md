# Minimal Chat Actions Plan

## Goal
Make the chat composer minimal on all screen sizes: show only text input + send in the main footer, and move advanced controls (model, mode, git action) into a dropdown.

## Confirmed Product Direction
- Advanced controls are hidden behind a dropdown on **all** breakpoints.
- Composer behavior contract stays the same (`onSubmit(text, mode, model)`).

## Files to Modify
- `src/components/chat/ChatFooter.tsx` (primary refactor)
- `src/components/chat/ChatView.tsx` (small spacing/container adjustments only if needed)
- `src/routes/_authed/chat.index.tsx` (verify first-run flow remains correct with new footer UI)

## Implementation Plan

### 1) Refactor `ChatFooter` into primary + advanced areas
**File:** `src/components/chat/ChatFooter.tsx`
- Keep visible by default:
  - textarea input
  - send button
  - one compact "options" trigger button
- Move into options dropdown:
  - model selector
  - mode selector (plan/build)
  - git quick action button
- Reuse existing primitives (`Popover`, `Select`) rather than introducing new UI infra.

### 2) Preserve existing behavior and state model
**File:** `src/components/chat/ChatFooter.tsx`
- Keep existing local state for `text`, `mode`, `model`.
- Keep keyboard behavior:
  - `Enter` submits
  - `Shift+Enter` inserts newline
- Keep submit/output contract unchanged so parent usage in `ChatView` and `chat.index` does not change.
- Keep disabled/loading gates consistent for both primary and advanced controls.

### 3) Accessibility and interaction quality
**File:** `src/components/chat/ChatFooter.tsx`
- Add `aria-label` and `aria-expanded` on options trigger.
- Keep touch target size >= 44px for send/options on mobile.
- Ensure dropdown is keyboard accessible and focus flow is intact (open, tab through controls, close).
- Add `aria-pressed` for mode toggle buttons if kept as segmented buttons inside dropdown.

### 4) Validate integration points
**Files:**
- `src/components/chat/ChatView.tsx`
- `src/routes/_authed/chat.index.tsx`
- Confirm both follow-up and first-session flows still pass correct `mode/model` from dropdown selections.
- Confirm no regressions in footer stacking/scroll behavior at small widths.

## Risk & Edge Cases
- Options dropdown clipped on narrow viewports.
- User submits while dropdown open (should still submit with current mode/model).
- Disabled or submitting state still allows accidental advanced action clicks.
- Missing/default model value handling when options are not opened.

## Verification Plan

### Automated
- `bun run check`
- `bun run test`

### Manual UX Checks
- Desktop + mobile: only input/send/options visible in composer.
- Open options and change model/mode, submit prompt, verify selected values are used.
- Trigger git quick action from options and verify it submits with selected mode/model.
- Validate `Enter` and `Shift+Enter` behavior.
- Validate disabled/loading state blocks send + options interactions appropriately.
- Validate no scrolling regression in `/chat/:sessionId` while footer is collapsed/expanded.

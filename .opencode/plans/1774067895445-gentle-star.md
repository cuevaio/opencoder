# Enforce Trigger OpenCode Plan Env

## Goal
- Always launch Trigger-run OpenCode sessions with `OPENCODE_EXPERIMENTAL_PLAN_MODE=1`.
- Keep existing `"plan" | "build"` mode selection behavior unchanged.

## Recommended Approach
1. Set the env var at the OpenCode startup choke point.
   - Modify `src/trigger/lib/opencode-server.ts` in `startOpenCodeServer(...)`.
   - Add `process.env.OPENCODE_EXPERIMENTAL_PLAN_MODE = "1"` immediately before `createOpencode(...)`.
   - Keep existing env prep logic (`PATH`, `OPENCODE_SERVER_PASSWORD`, `OPENCODE_SERVER_USERNAME`) intact.

2. Do not change plan/build routing behavior.
   - Leave mode flow untouched in:
     - `src/routes/api/agent/run.ts`
     - `src/routes/api/agent/continue.ts`
     - `src/trigger/run-session.ts`
   - Specifically preserve `agent: payload.mode` so UI mode selection still works.

3. Add focused regression coverage for launch env wiring.
   - Add `src/trigger/lib/opencode-server.test.ts`.
   - Mock `createOpencode` from `@opencode-ai/sdk/v2`.
   - Assert `OPENCODE_EXPERIMENTAL_PLAN_MODE` is `"1"` when `startOpenCodeServer(...)` launches OpenCode.

## Critical Files To Modify
- `src/trigger/lib/opencode-server.ts`
- `src/trigger/lib/opencode-server.test.ts` (new)

## Verification
1. Run checks:
   - `npm run check`

2. Run targeted test:
   - `npx vitest run src/trigger/lib/opencode-server.test.ts`

3. End-to-end validation:
   - Start one Trigger session with mode `"build"` and one with mode `"plan"`.
   - Confirm both still execute with current mode behavior (no forced mode changes).
   - Confirm OpenCode startup path in Trigger uses env var `OPENCODE_EXPERIMENTAL_PLAN_MODE=1` (via test assertion and/or startup log signal if added).

## Notes
- Do not use `trigger.config.ts` for this behavior; that file handles build image setup, while env must be set at runtime where OpenCode is started.

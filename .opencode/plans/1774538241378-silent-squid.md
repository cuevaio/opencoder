# Plan: Fix commit authorship to show the real user, not coder-agent[bot]

## Problem

Commits made by the agent show `coder-agent[bot]` as author instead of the authenticated user. Two causes:

1. **Email mismatch** — `session.user.email` (from Better Auth) may not match a verified GitHub email, so GitHub can't attribute the commit to the user and falls back to the OAuth App's bot identity.
2. **OpenCode overrides git config** — OpenCode likely sets `GIT_AUTHOR_NAME`/`GIT_AUTHOR_EMAIL` env vars or uses `--author`, overriding our `git config user.name/email`.

## Solution

Fetch the user's GitHub identity via the API, use the noreply email format that GitHub always recognizes, and set env vars that override everything.

### 1. New helper — `src/trigger/lib/github-identity.ts`

```ts
export interface GitHubIdentity {
  login: string;
  id: number;
  name: string | null;
}

export async function fetchGitHubIdentity(token: string): Promise<GitHubIdentity>
```

Calls `https://api.github.com/user` with the OAuth token. Returns `{ login, id, name }`. On failure, throws (the trigger task will fail with a clear error).

### 2. Resolve identity before clone — `src/trigger/run-session.ts`

Before calling `cloneRepo()`, fetch the GitHub identity and construct the proper git author:

```ts
const ghUser = await fetchGitHubIdentity(payload.githubToken);
const gitUser: UserIdentity = {
  name: ghUser.name || ghUser.login,
  email: `${ghUser.id}+${ghUser.login}@users.noreply.github.com`,
};
const clone = cloneRepo(payload.repoUrl, payload.githubToken, gitUser);
```

This replaces the current `{ name: payload.userName, email: payload.userEmail }`.

### 3. Set env vars in OpenCode server — `src/trigger/lib/opencode-server.ts`

Add a `gitUser: UserIdentity` parameter to `startOpenCodeServer()`. Set these env vars before `createOpencode()`:

```ts
process.env.GIT_AUTHOR_NAME = gitUser.name;
process.env.GIT_AUTHOR_EMAIL = gitUser.email;
process.env.GIT_COMMITTER_NAME = gitUser.name;
process.env.GIT_COMMITTER_EMAIL = gitUser.email;
```

These env vars override ALL other git identity mechanisms (git config, `--author` flag, OpenCode's own defaults). Belt-and-suspenders with the `git config` in `cloneRepo()`.

### 4. Update call site — `src/trigger/run-session.ts`

Pass `gitUser` to `startOpenCodeServer()`:

```ts
const { client, server } = await startOpenCodeServer(
  clone.cloneDir,
  abortController.signal,
  modelExecution.fullModel,
  payload.githubToken,
  gitUser,
);
```

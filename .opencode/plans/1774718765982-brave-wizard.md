# Plan: Fix overflow in Agent (task) tool row

## Goal
Prevent long `Agent (task)` descriptions from overflowing the tool-call box in chat while preserving current interaction and visual style.

## Recommended implementation
1. Update the header layout in `src/components/chat/ToolCall.tsx` (`GenericToolCall`) to use a constrained flex pattern:
   - Change header button from wrapped layout to a single-line flex container with `min-w-0`.
   - Introduce a text wrapper (`min-w-0 flex-1`) that contains title/subtitle text.
   - Keep status dot and chevron non-shrinking (`shrink-0`) so they remain visible.
2. Normalize text overflow behavior in that header:
   - Keep `truncate` for subtitle/title metadata fields that can be long.
   - Remove `[overflow-wrap:anywhere]` from elements using `truncate` to avoid conflicting behavior.
3. Leave `src/lib/tool-info.ts` unchanged (the `task` subtitle source is correct).
4. Do not change global `.scroll-region` overflow in `src/styles.css` unless local fix is insufficient; root cause is row-level flex sizing.

## Files to modify
- `src/components/chat/ToolCall.tsx`

## Why this approach
- Fixes the overflow at the component boundary where it originates.
- Matches existing resilient truncation patterns used elsewhere in the codebase.
- Minimizes regression risk by avoiding global overflow changes.

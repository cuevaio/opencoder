# Plan: Image Upload Support via Vercel Blob Storage

## Summary

Add image upload support to the chat composer. Users can attach images via file picker, clipboard paste, or drag-and-drop. Images are uploaded to Vercel Blob Storage and passed to OpenCode SDK as `FilePartInput` parts alongside text. Images display inline in user message bubbles with click-to-preview lightbox.

## Key Decisions

- **Upload strategy:** Server-side (file → app server → Vercel Blob). 4.5MB limit per image.
- **Input methods:** File picker button + clipboard paste + drag-and-drop.
- **Max images:** 10 per message.
- **Allowed formats:** `image/png`, `image/jpeg`, `image/gif`, `image/webp` (no SVG for security).
- **OpenCode SDK support:** Already has `FilePartInput { type: "file", mime, url }` — no SDK changes needed.

## Data Flow

### Upload Flow
```
User selects/pastes/drops image
  → Client validates type + size
  → Client shows local preview (URL.createObjectURL)
  → Client POSTs file to /api/upload/image (multipart/form-data)
  → Server validates auth + file, calls @vercel/blob put()
  → Server returns { url, mime, filename }
  → Client stores blob URL in pending images state
  → User presses Send
  → ChatFooter.onSubmit(text, mode, model, variant, imageUrls)
  → POST /api/agent/run (or /continue) with { ..., imageUrls }
  → Trigger.dev task builds parts: [{ type: "text", text }, ...imageUrls.map(→ FilePartInput)]
  → client.session.promptAsync({ parts })
```

### Display Flow
```
DB: session_events { event_type: "user-message", user_message_images: jsonb }
  → Electric SQL syncs to client
  → dbRowsToStreamEvents() → StreamEvent { type: "user-message", text, images }
  → buildDisplayItems() → DisplayItem with images
  → splitIntoTurns() → Turn { prompt, images, items }
  → SessionTurn renders image thumbnails in user bubble
  → Click → Dialog lightbox with full-size image
```

## Implementation Steps

### Step 1: Install dependency + env var

**`package.json`** — Add `@vercel/blob`
```bash
bun add @vercel/blob
```

**`.env.example`** — Add:
```
BLOB_READ_WRITE_TOKEN=
```

### Step 2: DB schema + migration

**`src/db/schema.ts`** — Add column to `sessionEvents`:
```ts
userMessageImages: jsonb("user_message_images"), // Array<{ url, mime, filename? }>
```

Run: `bun run db:generate && bun run db:push`

### Step 3: Create upload API endpoint

**`src/routes/api/upload/image.ts`** (new file)
- `POST /api/upload/image` — multipart/form-data with single `file` field
- Auth via `requireAuth(request)`
- Validate: file size ≤ 4.5MB, MIME in allowed list
- Upload via `put()` from `@vercel/blob` with path `uploads/{userId}/{timestamp}-{random}.{ext}`
- Return `{ url, mime, filename, size }`

### Step 4: Update type system

**`src/lib/session-types.ts`** — Add `images` to user-message StreamEvent:
```diff
-| { type: "user-message"; text: string }
+| { type: "user-message"; text: string; images?: Array<{ url: string; mime: string; filename?: string }> }
```

**`src/lib/display-items.ts`** — Add `images` to:
- `user-message` DisplayItem variant
- `Turn` type
- `buildDisplayItems()` — propagate `evt.images`
- `splitIntoTurns()` — carry images with turns

### Step 5: Update session converter

**`src/lib/session-converter.ts`**:
- Add `user_message_images` to `SessionEventRow` interface
- `dbRowsToStreamEvents()`: include `images` from `row.user_message_images`
- `sessionExportToStreamEvents()`: extract file parts from user messages → `images`

### Step 6: Update DB writer

**`src/trigger/lib/db-writer.ts`** — `writeUserMessage(text, images?)`:
- Accept optional `images` parameter
- Write `userMessageImages: images?.length ? images : null` to DB

### Step 7: Update Trigger.dev task

**`src/trigger/run-session.ts`**:
- Add `imageUrls` to `runSessionSchema`: `z.array(z.object({ url, mime, filename? })).default([])`
- Build `parts` array with text + file parts for `promptAsync()`
- Pass images to `dbWriter.writeUserMessage(payload.prompt, payload.imageUrls)`

### Step 8: Update API routes

**`src/routes/api/agent/run.ts`**:
- Accept `imageUrls` in request body
- Validate: max 10 images, Vercel Blob URL domain check
- Pass through to `tasks.trigger("run-session", { ..., imageUrls })`

**`src/routes/api/agent/continue.ts`**:
- Same changes as run.ts

### Step 9: Update Electric SQL shape proxy

**`src/routes/api/shapes/session-events.ts`**:
- Add `"user_message_images"` to the columns list

### Step 10: Update ChatFooter with image upload UI

**`src/components/chat/ChatFooter.tsx`**:
- Add `PendingImage` state: `{ id, url, mime, filename, status, localPreviewUrl, errorMessage? }`
- Add `imageUrls` parameter to `onSubmit` prop
- Add image upload function (shared by all input methods)
- Add `<input type="file" accept="image/*" multiple>` hidden ref
- Add paperclip/image button in bottom bar (next to advanced options)
- Add `onPaste` handler on Textarea for clipboard images
- Add `onDragOver`/`onDragLeave`/`onDrop` on wrapper div
- Add drag indicator visual (border highlight)
- Render pending images strip above textarea with thumbnails + remove buttons + upload status
- Disable send while any image is `"uploading"`
- On submit: filter ready images, pass to `onSubmit`, clear state

### Step 11: Update SessionTurn to render images

**`src/components/chat/SessionTurn.tsx`**:
- Accept `turn.images` from Turn type
- Render image thumbnails in user prompt bubble (above text)
- Each thumbnail: `<img>` with `h-20 w-20 object-cover rounded cursor-pointer`
- Click opens Dialog lightbox with full-size image
- Use existing `Dialog`/`DialogContent` from `src/components/ui/dialog.tsx`

### Step 12: Update route pages to thread imageUrls

**`src/routes/_authed/chat.index.tsx`** — `handleSubmit`: accept and forward imageUrls to POST body

**`src/routes/_authed/chat.$sessionId.tsx`** — `handleFollowup`: accept and forward imageUrls

**`src/components/chat/ChatView.tsx`** — Update `onFollowup` type to include imageUrls

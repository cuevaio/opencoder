# Mobile-First UI Rebuild Plan (Broader App Refresh)

## Locked Decisions
- Scope: broader app refresh (chat experience + shared app surfaces), not just one screen.
- Visual direction: minimal monochrome, screenshot-inspired.
- Approach: mobile-first defaults, then `md+` enhancements.

## Objectives
- Rebuild the UI so core flows feel deliberate on phones first (320px+), while preserving desktop efficiency.
- Unify duplicated navigation/sheet patterns and reduce ad-hoc styling in chat surfaces.
- Keep existing product behavior/data flow intact (session streaming, controls, actions), focusing on visual and layout system quality.

## Implementation Plan

### Phase 1 — Design System Foundation
1. Refresh global tokens and interaction primitives in `src/styles.css`:
   - Introduce a minimal monochrome surface ladder (`background`, `card`, `muted`, border strengths) and one restrained accent for status/action states.
   - Add motion tokens (duration/easing) and keep reduced-motion support.
   - Add safe-area helpers for top/bottom insets used by header/footer/sheets.
   - Define typography variables with a more intentional UI stack (while preserving readability and performance).
2. Normalize primitive ergonomics for touch-first UI:
   - `src/components/ui/button.tsx`
   - `src/components/ui/input.tsx`
   - `src/components/ui/textarea.tsx`
   - `src/components/ui/select.tsx`
   - Ensure baseline 44px interactive targets and consistent radius/spacing/ring behavior.
3. Upgrade sheet primitive for shared mobile navigation behavior in `src/components/ui/sheet.tsx`:
   - Tokenized width presets (remove per-screen hardcoded widths).
   - Consistent enter/exit motion timing.
   - Safe-area-aware content padding.

### Phase 2 — Shared Navigation and App Chrome
1. Consolidate mobile nav pattern between:
   - `src/components/app-header.tsx`
   - `src/components/chat/ChatMobileMenu.tsx`
   - Reuse shared sheet sizing/actions/spacing and reduce duplicate markup patterns.
2. Refresh top-level shell appearance for broader app consistency:
   - `src/routes/index.tsx`
   - `src/routes/sign-in.tsx`
   - `src/routes/_authed/dashboard.tsx`
   - Keep content structure intact, but apply new typographic rhythm, card treatment, and spacing hierarchy.

### Phase 3 — Chat Shell (Mobile-First Core)
1. Rework layout contract in `src/routes/_authed/chat.tsx`:
   - Keep desktop persistent sidebar at `md+`, mobile sheet below `md`.
   - Preserve behavior parity while reducing nested overflow risks.
   - Keep JS media query behavior aligned with CSS breakpoint semantics.
2. Align breakpoint source-of-truth in `src/hooks/use-media-query.ts` and consumers:
   - Keep one explicit mobile threshold contract and avoid drift between hook and utility classes.

### Phase 4 — Session + Composer Experience
1. Rebuild session page structure in `src/components/chat/ChatView.tsx` and `src/routes/_authed/chat.$sessionId.tsx`:
   - Cleaner header hierarchy (repo + session actions).
   - Better timeline spacing cadence.
   - More intentional status/cost metadata presentation.
2. Redesign composer controls for narrow widths in `src/components/chat/ChatFooter.tsx`:
   - Mobile-first stacked order: prompt first, actions secondary.
   - Keep Plan/Build + Model selector + Git quick action, but optimize visual priority and tap ergonomics.
   - Maintain Enter-to-send behavior and disabled/loading states.
3. Standardize turn/tool/question card anatomy:
   - `src/components/chat/SessionTurn.tsx`
   - `src/components/chat/ToolCall.tsx`
   - `src/components/chat/QuestionForm.tsx`
   - Use consistent border, radius, spacing, and state indicators.

### Phase 5 — New Session and Supporting Chat Views
1. Refresh new session route to match rebuilt chat shell:
   - `src/routes/_authed/chat.index.tsx`
   - `src/components/chat/RepoSelector.tsx`
2. Refresh session list/sidebar visual hierarchy and interaction polish:
   - `src/components/chat/SessionSidebar.tsx`
   - Improve active/hover/focus states, status chips, and spacing density.

### Phase 6 — Polish, Accessibility, and Performance
1. Accessibility pass across rebuilt surfaces:
   - focus-visible, icon button labels, heading order, contrast.
2. Motion pass using shared timing/easing only:
   - Avoid arbitrary durations and avoid heavy layout animations in long lists.
3. Scroll + keyboard reliability:
   - Validate iOS keyboard behavior with sticky/footer safe-area handling and timeline scrolling.

## Verification Plan
1. Run app and verify key routes manually:
   - `bun run dev`
   - Check `/`, `/sign-in`, `/dashboard`, `/chat`, `/chat/$sessionId`.
2. Mobile-first viewport checks (DevTools/MCP):
   - 320x568, 375x812, 390x844, 430x932.
   - Validate header, sheets, composer, safe-area insets, no clipped actions.
3. Tablet/desktop checks:
   - 768x1024, 1024x768, 1280x800.
   - Validate sidebar behavior, readable line length, and control density.
4. Interaction checks:
   - Open/close both mobile sheets, switch sessions, submit follow-up, toggle Plan/Build, change model, expand/collapse tool calls.
   - Confirm loading/error/idle/running states remain clear.
5. Accessibility and motion checks:
   - Keyboard-only navigation, focus rings, reduced-motion behavior, color contrast spot check.
6. Quality gates:
   - `bun run check`
   - `bun run test -- src/components/chat/ChatView.test.ts`
   - `bun run test -- src/components/chat/ChatFooter.test.ts`
   - If targeted tests are absent, run `bun run test` after adding/updating relevant tests.

### Step 2a: Model registry helpers

export const familyLabels: Record<string, string> = {
  openai: "OpenAI",
  anthropic: "Anthropic",
};

export function getModelsByFamily(): Record<string, ModelOption[]> {
  const groups: Record<string, ModelOption[]> = {};
  for (const option of modelOptions) {
    (groups[option.family] ??= []).push(option);
  }
  return groups;

**Routing indicator** per model (right-aligned, small text):
- `"direct"` (green-tinted) вҖ” user has the family's direct key (openai/anthropic)
- `"via gateway"` (muted) вҖ” no direct key, but has vercel/AI Gateway key
- `"no key"` (red-muted, item disabled) вҖ” no compatible key at all

Logic:
```ts
function getRoutingLabel(family: string, configuredKeys: Set<KeyProviderId>) {
  if (configuredKeys.has(family as KeyProviderId)) return { label: "direct", available: true };
  if (configuredKeys.has("vercel")) return { label: "via gateway", available: true };
  return { label: "no key", available: false };
}
```

Unavailable models are shown at `opacity-50` with `pointer-events-none`.
